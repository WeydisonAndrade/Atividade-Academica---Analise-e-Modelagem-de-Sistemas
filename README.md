# Atividade-Academica---Analise-e-Modelagem-de-Sistemas
Atividade acadêmica da matéria de Análise e Modelagem de Sistemas
Esta atividade teve como objetivo a análise dos requisitos do sistema, e a divisão dos requisitos em FUNCIONAIS e NÃO FUNCIONAIS, que foram posteriormente modelados para a linguagem UML.
O software de modelagem usado na atividade foi o Draw.IO
Foram especificados os Requisitos Funcionais e os Não Funcionais
Os requisitos foram modelados em linguagem UML, para Diagrama de Classes e Diagrama de Casos de Uso
